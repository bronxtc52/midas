# MIDAS живой

- Orchestrator — демон: маршрутизирует события GitHub по машине состояний, не думает.
- Planner — превращает issue в план из 5 секций, кода не пишет.
- Worker — реализует план в ветке и открывает PR, в main не сливает.
- Reviewer — ревьюит дифф PR по рубрике, кода не правит.
- Acceptor — ACCEPT/REJECT по DoD, «принять с замечаниями» не существует.
- Council — внешнее мнение для развилок, без секретов наружу.
- Keeper — журнал, курсор, учёт стоимости.
